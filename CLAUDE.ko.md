# CLAUDE.ko.md

## 1. 가장 먼저 할 일

- 개발자에게 응답할 때 기본 언어: 한국어

## 4. 작업 규칙

### 코드 생성 시

코드를 생성하기 전에 짧게 설명한다.

1. 왜 이 구조를 사용하는지
2. 더 단순한 방법이나 대안이 있는지
3. 이 코드의 트레이드오프
4. 개발자가 이해해야 할 핵심 개념

설명은 짧고 실용적으로 유지한다.

### 설계 결정 시

바로 추천안을 강요하지 않는다.

먼저 개발자의 선택 기준을 확인한다.  
선택 기준이 없다면 최대 3개의 질문만 한다.

추천을 할 때는 다음을 설명한다.

- 왜 이 선택이 기준에 맞는지
- 어떤 대안을 검토했는지
- 어떤 경우에 이 선택이 틀릴 수 있는지

### 범위 관리

현재 작업 범위를 벗어나는 기능을 추가하기 전에 `docs/PRD.md`를 확인한다.

v1이 동작하기 전에는 v2/v3 기능을 구현하지 않는다.

### 소통 방식

- 개발자에게는 한국어로 응답한다.
- 필요하면 인프라, Docker, AWS, CI/CD 비유를 사용한다.
- LLM 관련 주제는 백엔드 주제보다 조금 더 자세히 설명한다.
- 오버띵킹을 유발하지 않도록 설명은 짧게 유지한다.

## 5. 코딩 규칙

- 타입 힌트를 사용한다.
- 함수에는 docstring을 작성한다.
- 똑똑한 코드보다 명확하고 단순한 코드를 우선한다.
- 함수는 작고 설명 가능하게 유지한다.
- 한 파일이 300줄을 넘으면 분리를 검토한다.
- 주석은 “무엇을 하는지”보다 “왜 이렇게 했는지”를 설명할 때 사용한다.
- 중요한 결정에는 `WHY:` 주석을 사용한다.

예시:

```python
# WHY: v1에서는 튜닝 복잡도를 줄이기 위해 chunk size를 고정한다.
chunk_size = 800
